# 页面标题栏整体上移设计

## 目标

在不侵入 Android 系统状态栏安全区域的前提下，将首页、设置页、编辑提醒页和新建周期提醒页的标题栏及其下方内容整体向上移动 12.dp。

## 现状

三个页面组件均在根 `Column` 上依次使用：

1. `statusBarsPadding()` 提供系统状态栏安全距离。
2. `padding(20.dp)` 再为页面四周增加 20.dp 留白。

编辑提醒页和新建周期提醒页共用 `ReminderEditorScreen`，因此实际需要调整的页面组件为：

- `HomeScreen`
- `SettingsScreen`
- `ReminderEditorScreen`

标题栏偏低来自状态栏安全距离下方额外叠加的 20.dp 顶部留白。

## 方案

保留 `statusBarsPadding()`，不修改系统安全区域适配。将三个页面根容器的页面内边距统一为：

- 顶部：8.dp
- 左侧：20.dp
- 右侧：20.dp
- 底部：20.dp

把这些尺寸放入公共页面布局配置中，由三个页面共同使用，避免重复硬编码和页面位置不一致。

由于根容器顶部留白减少 12.dp，标题栏和标题栏下方的全部内容会作为整体上移 12.dp。标题栏内部尺寸、标题与内容间距以及按钮对齐关系保持不变。

## 不变范围

- 不移除或修改 `statusBarsPadding()`。
- 不修改标题栏高度。
- 不修改标题、返回按钮、设置按钮和加号按钮的内部对齐。
- 不修改页面内容间距、滚动行为和视觉样式。
- 不修改导航、提醒、编辑、设置、存储和无障碍业务逻辑。
- 不修改“时间到了”通知落地页。

## 验证

自动检查：

- Compose 布局测试确认公共页面顶部内边距为 8.dp。
- 首页、设置页和编辑页现有无障碍及交互测试继续通过。
- 完整单元测试和 Android 测试源码编译通过。
- Debug APK 构建成功。

手动检查：

- 首页设置按钮、标题和加号按钮保持垂直居中。
- 设置页返回按钮和标题保持垂直居中。
- 编辑页与新建页返回按钮和标题保持垂直居中。
- 不同屏幕尺寸和状态栏高度下，标题栏均位于状态栏安全区域下方。
- 标题栏与下方内容的间距保持原样，页面内容整体同步上移。
