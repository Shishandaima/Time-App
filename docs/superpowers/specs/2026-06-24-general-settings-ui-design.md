# 通用设置项 UI 设计

## 目标

在设置页“通用”分组内保留现有主题功能，并在主题选项之后依次新增字体大小、响铃时长和震动开关三项。本次仅实现 UI，不连接真实配置、提醒逻辑或持久化存储。

## 页面结构

“通用”卡片内按以下顺序显示：

1. 主题
   - 浅色
   - 深色
   - 跟随系统
2. 字体大小：标准
3. 响铃时长：10秒
4. 震动开关：开关控件

“关于”分组及设置页其他内容保持不变。

## 视觉样式

- 字体大小和响铃时长复用设置页现有 `SettingsValueRow` 行样式。
- 震动开关使用相同的行高、文字样式、左右边距和垂直对齐，并在右侧显示 Material `Switch`。
- 三个新增项位于现有“通用”卡片内，不新增嵌套卡片。
- 保留现有卡片圆角、背景、间距和主题颜色。
- 不改变主题单选项的布局和交互。

## 临时状态

- 字体大小右侧固定显示“标准”，不可点击。
- 响铃时长右侧固定显示“10秒”，不可点击。
- 震动开关默认开启。
- 震动开关只使用 `SettingsScreen` 内的 Compose 临时状态。
- 离开设置页后无需保存状态，再次进入可恢复默认开启。

## 无障碍

- 字体大小和响铃时长保留完整可读文本和值。
- 震动开关使用标准 Material `Switch` 语义，TalkBack 可识别其开启或关闭状态。
- 点击震动设置行或开关本身均可切换临时状态，点击区域保持充足。

## 不变范围

- 不修改主题切换功能。
- 不修改设置页以外的页面。
- 不新增配置模型、数据库字段、SharedPreferences、API 或提醒业务逻辑。
- 不修改路由、更新检查和 APP 版本显示。

## 验证

- Compose UI 测试确认“通用”分组显示四个设置标题，新增项顺序正确。
- 测试确认“标准”和“10秒”占位值可见。
- 测试确认震动开关默认开启并可在当前页面切换。
- 现有主题、返回按钮、更新检查和版本显示测试继续通过。
- 完整单元测试、Android 测试源码编译和 Debug APK 构建通过。
